2. 해당 xls파일을 xlsx로 바꾸기 (이름 바꾸기 말고 안전한 변환 방식 필요)
3. 택배관리 파일의 1행을 없앰
4. G열의 “즉납” 키워드를 없애고 “4400’을 입력
5. OKOSC프로그램에서 처방전검색(출력)-대기처방-처방전송일자(7일 전 ~ 오늘, 조제)로 검색하여택배목록을 누르면 나오는 엑셀파일에서(그냥 뜸, 이름은 통합 문서 n) “통합 문서 n”의 이름의 떠있는 엑셀 창 중, 가장 n의 값이 큰 창에서 정보를 가져옴.
6. 통합 문서 파일의 E ~ I열을 택배관리 파일의 A ~ E열이 끝나는 행부터 연속적으로 붙여넣음
7. 통합 문서의 A, C ~ D가 택배관리 파일의 K, L ~ M에 연속적으로 정보를 옮김
8. F ~ H열의 내용이 아래쪽으로 붙여넣은 행 만큼 채워짐(1, 4400, 한약)
9. 택배관리 파일의 E열과 D열에 모두 번호가 있고 번호가 다르다면, 둘 다 유지. 두 열 중 하나만 번호가 있다면 모두 D 열로 이동 후 E열을 공란으로 둠
10. 로컬환경 Element > 한약국 > 옹기한약서류 > 익산대장$$ 있는 파일에서(이하 익산대장. 해당 경로에 익산대장~인 파일이 하나만 있다고 보장됨)
11. L열에 연한 녹색 바탕을 가진 텍스트(이름, 전화번호, 주소의 형식이 보장됨)를 택배관리 파일에 이름은 A열, 전화번호는 D열, 주소는 C열에 정보가 없는 가장 아래쪽 행에 붙여넣음(형식 상관 없음) FGH열은 그대로 따라옴(1, 4400, 한약)
12. 사람이 검토 (오케이 하면 아래 13, 14번 실행)
13. 로젠택베(ldgis.ilogen.com/commom/html/main.html)사이트로 감 (크롬에 로그인된 창이 항상 열려있다고 보장됨)
14.  예약관리-주문등록/출력(복수건)으로 감
15. 파일열기에서 택배관리 파일을 업로드
16. 사람이 검토 (오류 열에 오류가 없는지)
